# Eliga - eFootball Tournament Platform

## Overview

Eliga is a Progressive Web App (PWA) for organizing and participating in eFootball tournaments. Users can create or join tournaments, manage matches, chat with other players, track statistics, and connect with friends. The app supports both French and English languages and targets African and European eFootball communities. Key features include:

- Tournament creation and management (pool/league formats, public/private)
- Match scheduling and score reporting
- Real-time messaging via WebSockets
- Friend system with groups
- Player statistics and standings
- Tournament reward/trophy system: top 2–3 finalists receive gold/silver/bronze trophies displayed in their profile (Stats page), with fireworks animation for recent champions
- Marketplace for buying/selling in-game items
- Web push notifications
- PWA installability on mobile and desktop
- **Multi-language support**: UI available in French (fr) and English (en) with LocaleProvider context; language preference saved to localStorage
- **Searchable country selector**: Registration form uses `SearchableCountrySelect` component with all 195+ world countries, sortable and searchable by user input
- **eLIGA Clips**: TikTok-style short video feed — players upload game techniques and match highlights. Videos hosted on Cloudinary (env vars: CLOUDINARY_CLOUD_NAME, CLOUDINARY_API_KEY, CLOUDINARY_API_SECRET). Tables: `clips`, `clip_likes`. Routes: GET/POST /api/clips, POST /api/clips/:id/like, POST /api/clips/:id/view, DELETE /api/clips/:id, GET /api/clips/user/:userId, GET /api/clips/upload-signature.

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### Frontend Architecture

- **Framework**: React 18 with TypeScript, using Vite as the build tool
- **Routing**: Wouter (lightweight client-side routing)
- **State/Data fetching**: TanStack Query (React Query v5) for server state; all API calls go through a centralized `apiRequest` helper in `client/src/lib/queryClient.ts`
- **UI Components**: shadcn/ui (New York style) built on Radix UI primitives, styled with Tailwind CSS
- **Forms**: React Hook Form + Zod for validation
- **Localization**: LocaleProvider in `client/src/lib/locale.tsx` manages language state (fr/en); translation strings stored in context; language toggle button on home page
- **Countries**: All world countries list in `client/src/lib/countries.ts` with search/sort utilities
- **Real-time**: WebSocket client in `use-websocket.ts` hook, with token-based auth and auto-reconnect
- **PWA**: Service worker (`client/public/sw.js`) with network-first strategy for API calls and assets, cache-first for static icons. Push notifications via Web Push API. Install prompts handled by `PWAInstallButton` and `PWAInstallPrompt` components
- **Auth**: Session-based authentication tracked via React Context (`client/src/lib/auth.tsx`); auth state comes from `/api/auth/me`
- **Layout**: Responsive — sidebar navigation on desktop, bottom tab bar on mobile

### Backend Architecture

- **Runtime**: Node.js with Express (TypeScript via tsx)
- **Entry point**: `server/index.ts` → registers routes from `server/routes.ts`, serves Vite in dev or static files in production
- **Session management**: `express-session` with `connect-pg-simple` for PostgreSQL-backed sessions
- **WebSocket**: `ws` library, token-based auth (one-time tokens issued via `/api/auth/ws-token`), per-user connection map for targeted messaging
- **Password hashing**: SHA-256 with a fixed salt (simple approach, not bcrypt)
- **Storage layer**: `server/storage.ts` defines an `IStorage` interface with all database operations implemented via raw SQL through the `pg` Pool
- **Push notifications**: `web-push` library using VAPID keys from environment variables

### Data Storage

- **Database**: PostgreSQL, accessed via `pg` Pool (`server/db.ts`) and Drizzle ORM for schema definition and migrations
- **Migrations**: SQL migrations auto-generated by Drizzle Kit and stored in `migrations/` directory (e.g., `migrations/0000_cool_abomination.sql`). Run migrations on database startup via `drizzle-kit push` or manual SQL execution
- **Schema** (`shared/schema.ts`): Shared between client and server. Key tables:
  - `users` — accounts with profile info, phone, country, region, admin flag
  - `friends` — friend relationships with pending/accepted status
  - `friendGroups` / `friendGroupMembers` — user-defined friend groups
  - `tournaments` — tournament config (type, game platform, visibility, pools, limits, sponsorship, elite/paid flags)
  - `tournamentParticipants`, `tournamentMatches`, `tournamentChat` — tournament activity
  - `messages` — direct messages between users
  - `notifications` — in-app and push notifications
  - `tournament_rewards` — rewards distributed to top 2–3 finalists when a tournament closes (only if >5 participants); stores position (1/2/3), badge (gold/silver/bronze), tournament level (standard/cotisation/sponsored/elite), and tournament name

### Authentication & Authorization

- Session cookies (PostgreSQL-backed) identify logged-in users
- `requireAuth` helper in routes guards protected endpoints
- Admin flag (`isAdmin`) on user record gates admin-only routes
- WebSocket connections are authenticated with short-lived tokens

### Build & Deployment

- Dev: `tsx server/index.ts` with Vite middleware for HMR
- Production build: Custom `script/build.ts` runs Vite for the client, then esbuild to bundle the server (with an allowlist of dependencies to bundle inline for faster cold starts)
- Output: `dist/public` (client) + `dist/index.cjs` (server)

## External Dependencies

| Dependency | Purpose |
|---|---|
| **PostgreSQL** | Primary database (via `DATABASE_URL` env var) |
| **Drizzle ORM** | Schema definition, migrations (`drizzle-kit push`) |
| **connect-pg-simple** | PostgreSQL session store |
| **web-push** | Web Push / VAPID push notifications (needs `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_EMAIL` env vars) |
| **Google Fonts** | Open Sans font loaded via CDN in `index.html` |
| **Replit plugins** | `@replit/vite-plugin-runtime-error-modal`, `@replit/vite-plugin-cartographer`, `@replit/vite-plugin-dev-banner`, `@replit/connectors-sdk` — dev tooling for Replit environment |
| **ws** | WebSocket server for real-time messaging |
| **@octokit/rest** | GitHub API client (present in dependencies, usage may be in unrevealed code) |
| **Recharts** | Charting library (via shadcn `chart.tsx`) |
| **date-fns** | Date formatting and manipulation |
| **nanoid / uuid** | ID generation |

### Environment Variables Required

- `DATABASE_URL` — PostgreSQL connection string
- `VAPID_PUBLIC_KEY` — VAPID public key for push notifications
- `VAPID_PRIVATE_KEY` — VAPID private key for push notifications
- `VAPID_EMAIL` — Contact email for VAPID (defaults to `mailto:admin@eliga.app`)
- `NODE_ENV` — `development` or `production`
